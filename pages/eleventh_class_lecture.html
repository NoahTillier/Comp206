Odin Project CSS notes 1

  1. Selectors
    1. Universal Selector
      1. The universal selector will select elements of every type. *{ color: purple; }
    2. Type selectors
      1. A type selector will select all elements of the given element type. div { color: white; } sets all
        elements to white
    3. Class selector
      1. You can create a class element with: <div class ="class">
      2. You can stype it with .class { color: red; }
      3. You can add multiple classes with a single space separated list. One class could change the text font, one could change the color, etc.
      4. The name of the class cannot start with a number
    4. ID selectors
      1. An element can only have one ID
      2. The element is selected based on its ID
      3. <div id="title">WORDS</div>
      4. #title { background-color: red; }
      5. The name of the ID can not start with a number
    5. Grouping Selector
      1. We can group similar declarations together with a comma
      2. .read, /n .unread { /n color: white; /n background-color: black; /n} /n.read{...} /n.unread{...}
    6. Chaining selectors
      1. If we have a class "subsection header" and a class "subsection preview", we can apply a unique style to exclusively subsection header (not subsection or header) with .subsection.header {...}
      2. You can replace the second dot with a "#" if the item is an ID instead of a class (allows for chaining IDs and classes).
  2. Properties
    1. color sets text color
    2. background-color sets background color
      1. Colors can be hexadecimal (#1100ff;), rgb (rgb(100,0,127);), or HSL (hsl(15, 82%, 56%))
      2. Colors can also be color names (ex. white) or transparent.
    3. Font
      1. font-family is a single value or comma-separated list of values.
        1. the font-family should have a list because it allows the browser to pick in descending order the first font that it supports.
        2. Examples are "Times New Roman" and serif
      2. font-size. Ex, font-size: 22px
      3. font-weight is the boldness of the text from 1 to 1000 (keyword bold is 700).
      4. text-align aligns the text within an element. Values could be center, left, right...
    4. Images
      1. images have height: and width: properties
      2. You can set either manually or adjust the size automatically with "auto"
        1. specifying a height and width prevents the page from jumping when the image loads in because space is reserved for it.
  3. How to add CSS to HTML
    1. External CSS
      1. Create an external CSS file and link it to the HTML head using the <link rel ="stylesheet" href="styles.css"&rt element. (Note that href is the actual link relative to the file location)
      2. The filename must be .css
    2. Internal CSS
      1. Write the CSS inside the file itself
      2. Inside the body of the <head&rt tag, place a <style> tag. The inside defines css.
    3. Inline CSS
      1. The CSS style is specified on the same line
      2. Avoid using except when writing a style for a single line
      3. Inline CSS overrides all other methods
      4. <div style="color:white...></div>

Odin Project: The Cascade

  1. The cascade
    1. Unexpected things happening in our HTML file are sometimes determined by the cascade.
    2. Specificity
      1. Inline styles have the highest specificity
      2. ID selects are the most specific selector
      3. Class selectors are the next most specific selector
        1. The more class selectors you have, the more specific a selector is. For example, .main .list is more selective than .subsection
      4. Type selectors are the least specific selector
      5. Specificity is used to determine tie-breakers
  2. Inheretance
    1. A family's descendants can inherit properties such as color, font-size, font-family, etc.
      1. To check whether a property is inherited or not, you can see the MDN documentation
    2. Inheritance looses to higher specificity.
  3. Final tie braeker
    1. At the end of the day, the last defined element wins the specificity wars.
List of all commands
Command Options Function
ls [options] [path] Shows a list of files in the current working directory
-a Includes hidden files ("." and "..", for example) to the list of files
-l Shows all listed files in long format; including permissions and when the file was created
mkdir [options] [name] Creates a new directory with the specified name
rmdir [options] [name] Removes an empty directory
-i Forces a confirmation before proceeding with the removal
-r Recursively visits files and subdirectories to preform the same command
-f Forces rmdir to act, overriding -i
cp [options] [file path] [destination path] Copies a file from one place to another specified directory
-i Forces a confirmation before proceeding with the copy
-r Recursively visits files and subdirectories to preform the same command
-f Forces rmdir to act, overriding -i
pwd Displays the current working directory
cd Changes the directory to one with a specified path
mv [options] [file path] [destination path] Moves a file from the specified path to a new specified path
-i Forces a confirmation before proceeding with the deletion
rm [options] [name] Deletes a non-directory file
-i Forces a confirmation before proceeding with the deletion
-r Recursively visits files and subdirectories to preform the same command
-f Forces rmdir to act, overriding -i
tar [options] Combines many files into a single file or extracts those files from a .tar file
-c Creates a new tar archive
-x Extracts a tar archive
-f Specifies the tar archive name
-v Activates verbose mode
-z Compresses the archive into a .tgz file
chmod [options][values] Changes the permissions (for user, group, and others) for a file or directory (read, write, execute)
=, -, + = sets permissions, - removes them, and + adds them (if they did not already exist)
r, w, x r stands for read, w stands for write, and x stands for execute
u, g, o, a u stands for the owner of the file, g stands for the group which the file belongs to, o stands for others, and a stands for all.
Binary settings Instead of using a bunch of options, chmod can be used to set everyone's permissions with a binary code. Each digit represents the permissions for user, group, and others in that order.
less [file path] Views files (or text) in the command prompt, allowing scrolling but without loading the whole file
head [options] Presents the first 10 lines of text
-n[number] specifies the number of lines to display
tail [options] Presents the last 10 lines of text
-n[number] specifies the number of lines to display
cut [options] [file] Extracts sections of text from each line of a file or text stream
-d Specifies a delimiter to extract; ex. ",", " ", " " would seperate by commas, spaces, and tabs respectively
-f If only one number is given, it extracts exclusively that column. Can be made to extract more columns by separating columns by commas. Can extract a range of columns by using a dash (ex. 1-3). Column counting starts from one.
wc [options] [file] Counts the number of words in the file. By default counts lines, words, and characters separated by a space before outputting file name
-c Counts the number of characters instead
-l Counts the number of lines
-w Counts the number of words
sort [options] [file] Sorts data in a file or standard input
-n Sorts by numeric keys (ex. a line starting with 5 will go before one starting with 25)
-r Sorts in descending order
-o Writes the sort to an output file
Uniq [options] [file] Filters out consecutive, repeated lines from a file
-c Prints number of occurences next to output
cat [file] Prints text to STDOUT
grep [options] [STRING] [File List] Outputs a result of pattern searching for STRING in File_List into STDOUT
-i Ignore case
-n displays the line number along with the line where a match was found
-c reports only a count of number of lines containing matches
-v Inverts, displaying only the lines which don't match
-l List file names where matches were found
